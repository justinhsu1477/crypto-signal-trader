# 虛擬貨幣跟單產品 — 法律風險分析

> 最後更新：2025-02
> 免責聲明：本文件僅供內部參考，不構成法律建議。產品化前請諮詢專業律師。

---

## 1. 產品定位與法律身份

### 我們是什麼？

| 身份類型 | 定義 | 監管等級 | 我們的狀態 |
|---------|------|---------|-----------|
| **交易所 (VASP)** | 持有用戶資金、提供交易撮合 | 最高 — 需要牌照 | ❌ 不是 |
| **投資顧問** | 提供具體買賣建議、收取顧問費 | 高 — 可能需要證照 | ⚠️ 灰色地帶 |
| **非託管交易工具** | 用戶自己的帳戶、自己的 API Key | 較低 — 免牌照 | ✅ 我們的定位 |

### 為什麼「非託管交易工具」是最安全的定位？

- **不持有用戶資金** — 資金始終在用戶自己的幣安帳戶中
- **不提供投資建議** — 我們只是執行用戶選擇追蹤的訊號
- **用戶控制 API Key** — 用戶可隨時撤銷、設定 IP 白名單、禁止提幣
- **業界先例** — 3Commas、Cornix、Cryptohopper 都採用相同定位

## 2. 台灣法規環境

### 2.1 現行法規

| 法規 | 狀態 | 對我們的影響 |
|-----|------|------------|
| VASP 洗錢防制登記 | 已施行，9 家業者完成登記 | 我們不持有用戶資金，可能不算 VASP |
| 虛擬資產服務法（專法） | 草案階段，目標 2025 年會期三讀 | 需持續關注，專法會更明確定義範圍 |
| 金管會監管範圍 | 目前針對交易所、幣商 | 交易工具/機器人目前不在明確監管範圍 |

### 2.2 VASP 登記制

依洗錢防制法第 6 條規定，提供虛擬資產服務之事業未完成登記者，不得提供服務，違反者有 2 年以下刑事責任。

**關鍵問題**：交易機器人/工具是否屬於「虛擬資產服務」？

- 目前 VASP 定義主要針對：交易所、幣商、錢包服務商
- 交易工具類服務**目前未被明確納入**，但也未被明確排除
- 專法通過後範圍可能擴大，需密切關注

### 2.3 法規變動風險

台灣虛擬資產服務法初期將允許 VASP 經營六項業務：
1. 虛擬資產交換
2. 虛擬資產平台
3. 虛擬資產移轉
4. 虛擬資產保管
5. 虛擬資產承銷
6. 虛擬資產借貸

「交易工具」不在上述六項中，但未來可能新增類別。

## 3. 風險等級分析

### 3.1 高風險

| 風險 | 說明 | 後果 |
|------|------|------|
| 被認定為「投資顧問」 | 行銷文案暗示保證獲利、推薦特定交易 | 可能違反證券投資信託及顧問法 |
| 系統 Bug 造成重大虧損 | 例如 SL 未掛上導致倉位爆倉 | 民事損害賠償訴訟 |
| API Key 洩漏/被盜 | 用戶的 API Key 從我們的系統洩漏 | 民事賠償 + 個資法責任 |

### 3.2 中風險

| 風險 | 說明 | 後果 |
|------|------|------|
| 訊號來源合法性 | 付費群組訊號若涉及市場操縱 | 轉發/執行者可能被牽連 |
| 法規變動 | 專法通過後被納入監管 | 需要補辦登記或調整服務 |
| 跨境法律問題 | 服務面向非台灣用戶 | 需遵守用戶所在地法規 |

### 3.3 低風險

| 風險 | 說明 | 後果 |
|------|------|------|
| 用戶跟單虧損索賠 | 有完善免責條款的情況下 | 訴訟成功機率很低 |
| 競爭對手法律挑戰 | 業界慣例已建立 | 不太可能發生 |

## 4. 業界法律防護實踐

### 4.1 3Commas 的法律框架（業界標竿）

**四層防線：**

1. **身份定義** — 明確聲明不是 broker、不是 financial advisor，只提供軟體工具
2. **全面免責** — 用戶承認承擔所有風險，包括可預見和不可預見的風險
3. **損害賠償上限** — 最大賠償 = 過去 24 個月訂閱費，免費用戶最多 EUR 100
4. **例外條款** — 只有「故意」或「重大過失」情況下才有責任

### 4.2 Cornix 的做法

- 定位為 Telegram/Discord 訊號自動執行工具
- 不提供交易建議，只執行用戶選擇追蹤的訊號群發出的指令
- 用戶自行設定風險參數（止損百分比、倉位大小等）

### 4.3 Crypto.com Trading Bots

- 明確聲明對使用 Trading Bots 產生的任何損失不承擔責任
- 強調交易機器人涉及高風險，過去表現不代表未來結果

## 5. 必要法律文件

### 5.1 服務條款 (Terms of Service)

必須包含以下條款：

```
[ ] 服務性質聲明 — 本服務為自動化交易工具，非投資建議或投資顧問服務
[ ] 風險承擔 — 用戶自行承擔所有交易風險及可能的財務損失
[ ] 免責聲明 — 對系統延遲、API 故障、網路中斷等技術問題不負責
[ ] 損害賠償上限 — 最大賠償金額不超過用戶已付訂閱費用
[ ] 無保證條款 — 不保證任何特定的交易結果或投資回報
[ ] 第三方服務 — 不對幣安等交易所的服務中斷、政策變更負責
[ ] 訊號來源 — 不對訊號提供者的準確性或合法性負責
[ ] 終止權利 — 保留隨時終止服務的權利
[ ] 管轄法院 — 約定管轄法院（建議台灣台北地方法院）
[ ] 資料處理 — API Key 的儲存、加密、使用範圍
```

### 5.2 風險揭示書 (Risk Disclosure)

用戶註冊時必須勾選確認：

```
[ ] 虛擬貨幣價格波動劇烈，可能在短時間內大幅下跌甚至歸零
[ ] 槓桿交易可能導致虧損超過初始投入的保證金
[ ] 自動交易系統可能因技術故障產生非預期的交易結果
[ ] 訊號提供者的歷史績效不保證未來表現
[ ] 用戶應自行評估財務狀況，僅使用可承受損失的資金進行交易
[ ] API Key 的安全管理責任歸用戶，建議禁用提幣權限並設定 IP 白名單
[ ] 市場流動性不足時，止損單可能無法以預期價格成交（滑點風險）
```

### 5.3 隱私政策 (Privacy Policy)

```
[ ] 收集的資料範圍（email、API Key 等）
[ ] API Key 加密儲存方式
[ ] 資料保留期限
[ ] 用戶刪除資料的權利
[ ] 不與第三方分享 API Key
```

## 6. 技術層面的法律防護

### 6.1 降低系統 Bug 風險

| 措施 | 目的 | 優先級 |
|------|------|--------|
| SL 必掛驗證 | 避免倉位裸奔 — 已完成 (P0) | ✅ 已完成 |
| 每日最大虧損限制 | 單日虧損超過 X% 自動停機 | ✅ 已完成 |
| 最大持倉限制 | 防止過度暴露 | ✅ 已完成 |
| 完整操作日誌 | 爭議時的證據保全 | ✅ 已完成 |
| API Key 加密存儲 | 避免明文洩漏 | 🔶 產品化前必做 |
| 禁止提幣權限驗證 | 只接受無提幣權限的 API Key | 🔶 產品化前必做 |
| 自動化測試覆蓋 | 確保核心交易邏輯正確 | 🔶 持續加強 |

### 6.2 API Key 安全要求

產品化時必須要求用戶的 API Key：
- ❌ 禁用提幣 (Withdraw) 權限
- ✅ 只開啟合約交易 (Futures) 權限
- ✅ 設定 IP 白名單（綁定我們的伺服器 IP）
- 系統應在接收 API Key 時驗證其權限設定

## 9. 結論

| 問題 | 結論 |
|------|------|
| 會有法律問題嗎？ | 以「非託管交易工具」定位，風險**可控但不為零** |
| 用戶賠錢會告嗎？ | 有完善免責條款 + 風險揭示書，成功索賠機率很低 |
| 需要 VASP 牌照嗎？ | 目前不需要，但專法通過後需重新評估 |
| 最大法律風險？ | 1) 系統 Bug 造成非預期虧損 2) 被認定為投資顧問 |
| 業界怎麼做的？ | 3Commas/Cornix 都用「工具不是顧問」定位 + 完整免責 |
| 建議？ | 產品化前請律師寫正式法律文件，持續關注法規變動 |
